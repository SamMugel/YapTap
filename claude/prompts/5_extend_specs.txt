Clarifying questions                                                                                                                                   
                                                                                                                                                         
  1. How does Rust find config/prompts/ when not running from the repo root?                                                                             
                                                                                                                                                         
  The spec says config/prompts/ is "relative to the project root," which works while developing in the repo. But the roadmap says the binary is          
  installable via cargo install. Once installed, yaptap runs from wherever the user's terminal is — config/prompts/ won't exist relative to CWD.         
                                                                                                                                                         
  Three options:                                            
  - (a) CWD-relative — only works from the repo root; fine if this is a dev-only tool for now
  - (b) Binary-relative — Rust resolves config/prompts/ relative to the binary's own path (e.g. std::env::current_exe())                                 
  - (c) Hardcoded absolute path the user configures — too complex for phase 2                                           
                                                                                                                                                         
  Which do you want?                                                                                                                                     
                                                                                                                             
  ---                                                                                                                                                    
  2. What happens on SIGINT (Ctrl-C) during LLM streaming?  
                                                                                                                                                         
  The spec covers SIGINT during recording (delete temp file, exit 130). But what if the user presses Ctrl-C while the LLM is generating output? Should
  it:                                                                                                                                                    
  - Kill the llm.py subprocess immediately and exit 130?    
  - Let the current streamed output stand, print a newline, then exit 130?
